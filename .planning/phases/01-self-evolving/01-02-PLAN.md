    ##### `learn suggest`
    Usage: `./piharness.sh learn suggest`
    Analyzes task-history.json for patterns:
    1. Groups tasks by keyword similarity (simple: extract keywords from descriptions)
    2. For clusters of 2+ tasks with no matched_skill, suggests creating a skill
    3. Display suggestions:
    ```
    Suggestions for new skills:
    ───────────────────────────────────────────
    web-builder (2 similar tasks)
      Trigger pattern: "web page", "website", "html"
      Run: piharness skill new web-builder --pattern "web page" --desc "Build web pages"
    ```

    Implementation for clustering — simple approach:
    - Strip common words (build, a, the, create, for, with, using, make)
    - Extract remaining significant words as keywords
    - Group tasks sharing 2+ keywords
    - For each group of 2+, suggest skill creation
    - Use python3 for the analysis since JSON parsing and text processing is cleaner

    Example python3 snippet for clustering:
    ```python
    import json, os, re
    STOP_WORDS = {'a','an','the','for','to','in','on','at','with','by','from','and','or','of','is','are','was','were','be','been','being','have','has','had','do','does','did','but','not','what','which','who','whom','this','that','these','those','build','create','make','using','into','out','up','down','new','old'}
    hist = json.load(open(os.path.expanduser('$TASK_HISTORY')))
    keywords = {}
    for t in hist.get('tasks',[]):
        words = set(re.findall(r'[a-zA-Z][a-zA-Z0-9]{2,}', t['description'].lower())) - STOP_WORDS
        keywords[t['id']] = words
    # Simple clustering: group tasks that share 2+ keywords
    clusters = []
    used = set()
    for t1 in hist['tasks']:
        if t1['id'] in used: continue
        cluster = [t1]
        used.add(t1['id'])
        for t2 in hist['tasks']:
            if t2['id'] in used: continue
            shared = len(keywords[t1['id']] & keywords[t2['id']])
            if shared >= 2:
                cluster.append(t2)
                used.add(t2['id'])
        if len(cluster) >= 2:
            all_kw = set()
            for t in cluster: all_kw |= keywords[t['id']]
            clusters.append({'tasks': [t['id'] for t in cluster], 'count': len(cluster), 'keywords': list(all_kw)[:5]})
    # Print suggestions
    for c in clusters:
        name = '-'.join(c['keywords'][:2])
        print(f'{name} ({c["count"]} similar tasks)')
        print(f'  Keywords: {\", \".join(c[\"keywords\"])}')
        print(f'  Run: piharness skill new {name} --pattern \"{c[\"keywords\"][0]}\" --desc \"Auto-suggested from {c[\"count\"]} tasks\"')
    ```

    Add the analysis to the `learn suggest` handler. If no clusters found, print "No patterns detected yet — keep working!".

    New behavior: instead of erroring, auto-close the oldest workers until only MAX_WORKERS - 1 remain (leaving room for the new one).

    Implementation:
    ```bash
    worker_count=$(wc -l < "$REGISTRY" | tr -d ' ')
    if [[ "$worker_count" -ge "$MAX_WORKERS" ]]; then
      close_count=$(( worker_count - MAX_WORKERS + 1 ))
      echo "Auto-closing $close_count old worker(s) to make room..."
      # Close oldest first (entries are chronological in registry)
      while IFS=$'\t' read -r surf lbl cwd wt br start_epoch role; do
        [[ $close_count -le 0 ]] && break
        # Skip if this is the only surface with orchestrator role
        bash "$SCRIPT_DIR/piharness.sh" close "$surf" >/dev/null 2>&1 || true
        echo "  Closed $surf ($lbl)"
        close_count=$(( close_count - 1 ))
      done < "$REGISTRY"
    fi
    ```

    Also add a `--max-workers N` flag to the `spawn` command that temporarily overrides MAX_WORKERS for that invocation:
    ```bash
    while [[ $# -gt 0 ]]; do
      case "$1" in
        --cwd)      cwd="$2";   shift 2 ;;
        --label)    label="$2"; shift 2 ;;
        --worktree) use_worktree=true; shift ;;
        --branch)   branch="$2"; shift 2 ;;
        --role)     role="$2";  shift 2 ;;
        --max-workers) MAX_WORKERS="$2"; shift 2 ;;  # NEW
        *) echo "Unknown arg: $1" >&2; exit 1 ;;
      esac
    done
    ```

    This ensures spawning never fails due to max workers — it just makes room.

    ##### `skill extract <surface> [--name NAME]`
    Usage: `./piharness.sh skill extract surface:5 --name my-new-skill`
    Extracts a skill from a completed worker's output:
    1. Gets the worker's output file using `_outfile <surface>`
    2. Reads the output, analyzes it to extract a description and trigger patterns
    3. Creates a new skill scaffold with the NAME (or auto-generates from content)
    4. If no --name, auto-generate: take first 3 significant words from output, join with hyphens
    5. Uses python3 to parse the output and generate appropriate trigger patterns
    6. Registers the skill

    ```bash
    extract)
      surface="$1"; shift
      name=""; trigger=""; desc=""
      while [[ $# -gt 0 ]]; do
        case "$1" in
          --name) name="$2"; shift 2 ;;
          --trigger) trigger="$2"; shift 2 ;;
          --desc) desc="$2"; shift 2 ;;
          *) echo "Unknown: $1" >&2; exit 1 ;;
        esac
      done
      [[ -z "$surface" ]] && { echo "Usage: piharness skill extract <surface> [--name N] [--trigger T] [--desc D]" >&2; exit 1; }
      outfile="$(_outfile "$surface")"
      [[ ! -f "$outfile" ]] && { echo "No output for $surface" >&2; exit 1; }

      # If no name, auto-generate from output content
      if [[ -z "$name" ]]; then
        name=$(grep -v -E '__PIHARNESS_DONE__|__PIHARNESS_ERROR__' "$outfile" 2>/dev/null \
          | tr ' ' '\n' | grep -oE '[a-zA-Z][a-zA-Z0-9]{3,}' | tr '[:upper:]' '[:lower:]' \
          | sort | uniq -c | sort -rn | head -5 | awk '{print $2}' | tr '\n' '-' | sed 's/-$//')
        [[ -z "$name" ]] && name="extracted-$(date +%s)"
      fi
      [[ -z "$trigger" ]] && trigger="$name"
      [[ -z "$desc" ]] && desc="Auto-extracted from worker $surface"

      # Create skill
      bash "$SCRIPT_DIR/piharness.sh" skill new "$name" --pattern "$trigger" --desc "$desc"
      echo ""
      echo "Extracted skill: $name"
      echo "  Edit the template at: $SKILLS_DIR/$name/template.md"
      echo "  Edit the description at: $SKILLS_DIR/$name/SKILL.md"
      ;;
    ```
