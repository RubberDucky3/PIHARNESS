    After the "Core Rule" section and before "Workflow", add:

    ```markdown
    ## Skill-Aware Orchestration (Self-Evolving System)

    PIHARNESS is a **self-evolving system** that automatically learns from its work.
    Skills are stored in `~/.piharness/skills/` and managed via `./piharness.sh skill`.

    ### Before every task:
    1. Run `./piharness.sh skill list` to check installed skills
    2. Run `./piharness.sh learn suggest` to check for pattern-detection suggestions
    3. If a matching skill exists, `./piharness.sh skill show <name>` to load its template
    4. Incorporate skill patterns into the worker prompts

    ### After every task:
    1. Run `./piharness.sh learn track "<task>" --surface <surface> --outcome <success|failure>`
    2. If the task was repetitive or used a novel pattern, consider:
       `./piharness.sh skill extract <surface> --name <skill-name>`

    ### Pane management:
    - Before spawning workers, run `./piharness.sh status` to check current count
    - Spawn will auto-close excess workers (over `$PIHARNESS_MAX_WORKERS`)
    - Use `./piharness.sh close <surface>` to clean up
    - Max 3 concurrent workers enforced by auto-close

    ### Self-evolution loop:
    ```
    Task arrives → Check skills → Match? → Execute with skill pattern
                                       → No match? → Execute ad-hoc
                                       → After success → Extract pattern as new skill
                                       → After 3+ similar tasks → Auto-suggest
    ```
    ```

    Also update the "Key commands reference" section to include:
    ```bash
    ./piharness.sh skill list                          # list installed skills
    ./piharness.sh skill show <name>                   # load skill details
    ./piharness.sh skill extract <surface> --name N    # create skill from work
    ./piharness.sh learn track <desc> --surface S      # log task for pattern detection
    ./piharness.sh learn suggest                       # check for skill suggestions
    ```

    And under "What NOT to do" add:
    - Do not skip skill registry checks — always check before planning
    - Do not forget to track tasks with `learn track` after completion

<task type="auto">
  <name>Task 3: Add `skill test` command with worktree isolation</name>
  <files>
    piharness.sh
  </files>
  <action>
    Add a `skill test` verb to the skill subcommand that tests a skill by:
    1. Creating a temporary worktree
    2. Running the skill's template against a test input
    3. Collecting results
    4. Cleaning up the worktree

    ```bash
    test)
      name="$1"; shift
      test_input=""
      while [[ $# -gt 0 ]]; do
        case "$1" in
          --input) test_input="$2"; shift 2 ;;
          *) echo "Unknown: $1" >&2; exit 1 ;;
        esac
      done
      [[ -z "$name" ]] && { echo "Usage: piharness skill test <name> [--input '...']" >&2; exit 1; }

      # Find skill
      skill_dir=$(python3 -c "
    import json, os
    reg = json.load(open(os.path.expanduser('$REGISTRY')))
    for s in reg['skills']:
        if s['name'] == '$name':
            print(s['path'])
            exit(0)
    exit(1)
    ") || { echo "Skill '$name' not found" >&2; exit 1; }

      template_file="$skill_dir/template.md"
      [[ ! -f "$template_file" ]] && { echo "No template.md for skill '$name'" >&2; exit 1; }

      # Create temporary worktree for testing
      test_branch="skill-test/$name-$(date +%s)"
      test_wt="$WORKTREES_DIR/$test_branch"
      git -C "$REPO_DIR" worktree add -B "$test_branch" "$test_wt" HEAD >/dev/null 2>&1 || {
        echo "Warning: worktree creation failed, testing in temp dir"
        test_wt=$(mktemp -d)
        cleanup_wt=true
      }

      echo "Testing skill: $name"
      echo "  Worktree: $test_wt"
      echo "  Template: $(cat "$template_file" | head -5 | tr '\n' ' ')"
      echo ""
      echo "  Test input: ${test_input:-<none>}"
      echo ""

      # Build prompt from template + input
      if [[ -n "$test_input" ]]; then
        prompt="$(cat "$template_file")\n\nTest input: ${test_input}\n\nApply the skill pattern to this task."
      else
        prompt="$(cat "$template_file")\n\nExecute this skill according to its template."
      fi

      # Run the test — spawn a temp worker, send the prompt, collect results
      echo "  Spawning test worker..."
      test_surface=$(bash "$SCRIPT_DIR/piharness.sh" spawn --label "skill-test-$name" --cwd "$test_wt" 2>&1 | grep -oE 'surface:[0-9]+' | head -1)

      if [[ -n "$test_surface" ]]; then
        bash "$SCRIPT_DIR/piharness.sh" task "$test_surface" "$prompt" >/dev/null 2>&1
        bash "$SCRIPT_DIR/piharness.sh" wait "$test_surface" --timeout 120 >/dev/null 2>&1 || true
        echo "  === Test Output ==="
        bash "$SCRIPT_DIR/piharness.sh" collect "$test_surface" 2>/dev/null || echo "(no output)"
        echo "  === End ==="
        bash "$SCRIPT_DIR/piharness.sh" close "$test_surface" >/dev/null 2>&1
      else
        echo "  Warning: could not spawn worker. Running prompt directly:"
        echo "$prompt"
      fi

      # Cleanup worktree
      if [[ -n "$test_wt" ]]; then
        git -C "$REPO_DIR" worktree remove --force "$test_wt" 2>/dev/null || rm -rf "$test_wt"
        git -C "$REPO_DIR" branch -D "$test_branch" 2>/dev/null || true
      fi

      echo ""
      echo "Test complete. Review output above."
      ;;
    ```

    Add this verb to the existing `skill)` case block.

