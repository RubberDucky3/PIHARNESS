PIHARNESS

Pi Worker Orchestrator for Claude Code

Spawn Pi workers, send tasks, collect output, compare results, and auto-learn from repetitive patterns — all from cmux panes alongside your orchestrator session.

./piharness.sh spawn --label worker-a ./piharness.sh task surface:5 "build X" ./piharness.sh skill list

What is PIHARNESS?

PIHARNESS is a bash-based worker orchestration harness that runs alongside your Claude Code session in cmux. It manages Pi worker panes so the orchestrator never writes code directly.

Plan

Break work into parallel slices

Execute

Spawn workers, send tasks in parallel

Learn

Auto-detect patterns, extract skills

Core Workflow

1. PLAN → 2. SPAWN → 3. TASK → 4. WAIT → 5. COLLECT → 6. COMMIT 
# 1. Check current state
./piharness.sh status

# 2. Spawn workers in parallel (max 3)
./piharness.sh spawn --label impl-structure --worktree
./piharness.sh spawn --label impl-logic --worktree

# 3. Send tasks
./piharness.sh task surface:5 "Build the HTML structure"
./piharness.sh task surface:6 "Implement the API logic"

# 4. Wait for completion
./piharness.sh wait surface:5 --timeout 300
./piharness.sh wait surface:6 --timeout 300

# 5. Collect and compare
./piharness.sh collect surface:5
./piharness.sh collect surface:6
./piharness.sh compare surface:5 surface:6

# 6. Apply best result and commit
# (orchestrator reviews output, applies to repo)

Command Reference

Spawn / Register

spawn [--cwd DIR] [--label L] [--worktree] [--branch B] [--role ROLE]

Create a new Pi worker pane. First worker splits right, subsequent ones split down. Use --worktree for isolated git branches.

use <surface> [--label L]

Register an existing cmux pane as a named worker.

Task Lifecycle

task <surface> <prompt>

Send a fire-and-forget task to a Pi worker.

auto <surface> <prompt>

Task with automatic runtime failover on error. Recommended for production use.

wait <surface> [--timeout N]

Block until the worker task completes (default 300s).

collect <surface>

Get the complete output from a finished task.

peek <surface>

Read partial output while the worker is still running.

Skill Management (Self-Evolving)

skill list

List all installed skills with version, usage count, and description.

skill show <name>

Show skill details + increments usage counter.

skill new <name> [--pattern P] [--desc D]

Create a new skill scaffold with SKILL.md and template.md.

skill extract <surface> [--name N]

Create a skill from a completed worker's output.

skill suggest

Get skill suggestions from task history patterns.

learn track <desc> [--surface S] [--outcome O]

Log a task for pattern detection.

learn history [--limit N]

View recent task history.

learn suggest

Analyze history for repeated patterns and suggest new skills.

Observability & Comparison

status

Dashboard: surface, label, status, elapsed, role, runtime, last line.

screen <surface>

Live terminal capture of the worker pane.

list

Full table of registered workers with metadata.

compare <s1> <s2>

Side-by-side output comparison.

diff <s1> <s2>

Git diff between worktree branches.

Cleanup

close <surface> [--keep-worktree]

Exit a worker pane and clean up.

clean

Reset all state: outputs, logs, registry.

Self-Evolving Skill System

PIHARNESS learns from repetitive work. Tasks that repeat 3+ times get extracted into reusable skills with trigger patterns, templates, and auto-suggestion.

Skill Lifecycle

Task arrives → Check ~/.piharness/skills/ for matching patterns
  → Match? → Reference skill in worker prompt
  → No match? → Execute normally
  → If task repeats 3+ times → Extract as skill

# Extract a skill from worker output
./piharness.sh skill extract surface:5 --name my-pattern

# The system auto-suggests via keyword clustering
./piharness.sh learn suggest
./piharness.sh skill suggest

Role-Based Pipeline

For complex features, spawn workers with specific roles and run a full implement→test→review cycle.

# Spawn role-specific workers
./piharness.sh spawn --role implementer --worktree --label impl-1
./piharness.sh spawn --role tester      --worktree --label test-1
./piharness.sh spawn --role reviewer    --worktree --label review-1

# Run automated pipeline
./piharness.sh pipeline "Deploy Express API" --rounds 3

# Or orchestrate manually
./piharness.sh orchestrate surface:7 "Review and merge changes"

Runtime Chain

When a model hits rate limits or errors, the runtime chain automatically fails over to the next available runtime:

pi:kilo-auto/free → pi:stepfun/step-3.7-flash:free → opencode:claude-haiku-4-5 → ollama:qwen2.5-coder:7b → claude:default

Override with PIHARNESS_RUNTIMES env var. Use ./piharness.sh runtimes to inspect.
